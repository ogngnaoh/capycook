# Design-Review Memo — CapyCook / Dish Development Workbench (v0.2 → v0.3)

> Synthesis of five research artifacts (architecture, culinary, product/UX, market-feasibility, adversarial critique) against DESIGN.md v0.2. Calibration preserved: **portfolio flagship** for big-tech SWE/systems internship review — engineering signal + intellectual honesty + solo-semester feasibility. Market defensibility is a non-goal. Every correction and recommendation below traces to a named finding; all source URLs are reused from the research files (none invented).

## Executive read

The design is fundamentally sound — the four load-bearing pillars (gated move/state machine, versioned branchable draft, deterministic/generative split, eval harness) all read as real systems engineering, and the epistemic posture ("measure, don't assert") is unusually mature for a portfolio piece. But the doc currently **bets its identity on its weakest leg**: the README headline stakes the pitch on "does flavor grounding beat a strong LLM?" — a hypothesis whose positive case is most-likely a tautology (a calculator beats an LLM at arithmetic, and that win belongs to the deterministic split, not grounding) and whose contested case (flavor-pairing) is built on a 2011 hypothesis that is empirically shaky, Western-cuisine-specific, and *inverted* for East/South Asian cooking. Two P0 items (an over-labeled "event-sourced" draft and a 3-source hybrid-retrieval pipeline wired as a hard dependency of the hero) are scoped at production fidelity inside a 5-week window, and the doc carries unearned pattern labels ("blackboard," "CQRS") plus a fabricated-as-fact citation that a knowledgeable reviewer will catch. The fix is reframing, not rebuilding: make process metrics + methodology the hero, demote the grounding number to an honest footnote, relabel the version chain truthfully, push live retrieval to P1, and add a calibrated safety gate.

## The hero-artifact reframe

**Why.** The critique's central honesty crack (its "internal contradictions" §): you cannot call food "substrate" while staking the flagship deliverable on a food-science result. Worse, the ablation's most-likely *positive* outcome is a tautology — the clean correctness win is USDA arithmetic/entity resolution, which the design already routes through deterministic functions, not the grounded LLM arm (culinary §5; critique [High]). Meanwhile the product-UX steelman locates the real edge over ChatGPT in **durability/resumability + trustable numbers** (product §4) — the engineering shell, not flavor grounding. So the hero as written measures a hypothesis whose win is a tautology, whose contested half is most-likely null, and which does not measure what actually differentiates the product.

**What it becomes.** The hero is the **engineering spine + the eval *methodology itself*** — neither depends on flavor-pairing being true. The headline metrics become process-level and sign-robust: **claim-provenance / hallucination rate** and **gate accept/edit/reject dynamics** (already listed in §9.3), plus inter-rater reliability and a versioned benchmark set. The 3-arm grounding ablation stays (it aids null-interpretability — critique caught that a 2-arm cut can't distinguish "grounding doesn't help" from "this retrieval impl is weak"), but it is now an **openly-hedged supporting finding**, with the grounding number a footnote, not the headline. README rewritten so methodology + intellectual honesty lead. The thesis then survives any ablation outcome.

## Top risks

- **[High] Hero measures the wrong thing.** Eval headline staked on a tautological/likely-null hypothesis instead of the product's real edge. → critique [High]; culinary §5; product §4. *Fix:* hero reframe above.
- **[High] Live-literature retrieval is the timeline-killer, wired as a hard dependency of the hero.** A query→fetch→citation-extraction→ranking→fusion pipeline is a multi-week subsystem; a slip cascades into the eval slipping. → architecture [High]; market feasibility; critique [High]. *Fix:* P0 grounded arm = FlavorGraph + USDA/FoodOn only; live literature → P1 layered on a working ablation.
- **[High/Critical] No safety section** for a system proposing novel combinations to home cooks — botulism (anaerobic oil infusion), undercooking, allergen-substitution blindness; discovery mode is *most* dangerous yet treated as lighter-weight. → culinary §4; critique [Med]. *Fix:* calibrated deterministic safety-gate (blocklist + disclaimer), not an FDA rules engine.
- **[Med] Buzzword density reads as resume-driven design.** "Blackboard" is unearned (P0 has one generative agent, not multiple opportunistic knowledge sources); "CQRS"/"event-sourced" overclaim. Any aspirational label taints the rest under senior scrutiny. → critique [Med]; architecture over-engineering. *Fix:* drop "blackboard" and "CQRS"; claim only patterns built to depth.
- **[Med] Approval fatigue.** A six-verb gate at *every* move, with no granularity distinction, is the single best-documented failure mode of this interaction model. → product §3 (DeepMind "AI Agent Traps", Franklin et al. 2025). *Fix:* pull a minimal autonomy dial (auto-advance deterministic moves only) into P0.
- **[Med] Citation that a reviewer will catch.** The FoodPuzzle "live > static" claim is used as the §13 rationale for hybrid retrieval but is fabricated-as-fact. → market §4; critique. *Fix:* corrections table above.
- **[Low] Undeclared durable-execution gap; undecided transport.** "Interruptible/checkpointed" implies guarantees not provided; "SSE or WebSockets" reads as undecided. → architecture [Med]/[Low]. *Fix:* scope to single-process in-session cancel as a stated choice; lock SSE + separate cancel endpoint.

## Prioritized recommendations

1. **Reframe the hero to process metrics + methodology; demote grounding to a hedged supporting result** (rewrite §17 and §9). → critique [High]; culinary §5; product §4.
2. **Relabel the draft a git-style snapshot+diff+branch version chain; reserve true event-sourcing for the move/gate event log only.** → architecture [High]; critique "contradiction resolved" — https://dev.to/devcorner/git-as-an-event-sourced-system-understanding-event-sourcing-through-git-271p
3. **Move live-literature retrieval to P1; P0 grounded arm = FlavorGraph + USDA/FoodOn.** → critique [High] #2; market rec 4.
4. **Add a deterministic SAFETY-GATE service** (anaerobic-preservation blocklist, min cook-temps, allergen check vs `constraints.allergens`) + disclaimer; give discovery mode the same gate. → culinary §4, recs 2 & 5 — https://ask.usda.gov/s/article/Can-you-get-botulism-from-garlic-in-oil
5. **Make flavor suggestions cuisine-aware and present FlavorGraph as a contested signal.** → culinary §1, rec 4 — https://arxiv.org/abs/1502.03815
6. **Fix the FoodPuzzle / FoodSky / Macaron / FoodKG citations; soften the ChatGPT and "no tool" claims.** → market recs 1, 2, 3, 5.
7. **Drop "blackboard" and "CQRS"; lock SSE + separate cancel endpoint; scope streaming to single-process in-session cancel as a deliberate choice.** → critique [Med]; architecture recs 3 & 6 — https://www.diagrid.io/blog/checkpoints-are-not-durable-execution-why-langgraph-crewai-google-adk-and-others-fall-short-for-production-agent-workflows
8. **Cite Magentic-UI as architectural prior art for the HITL interaction model** in §4/§13. → product rec 3 — https://arxiv.org/abs/2507.22358
9. **Promote tracing to the explicit P0 list; add inter-rater spot-check (15–20% double-label) + version the benchmark set.** → architecture gaps & rec 5.
10. **Pull a minimal autonomy dial into P0; add idempotency for "accept," reconcile "take-over" with the version log, state the gate-at-the-counter boundary, one sentence on steering-conversation persistence.** → product recs 1, 2, 5; architecture gaps.

## Open questions for the author

- **Benchmark labeler(s):** who double-labels the inter-rater subset — you alone across two sittings, or a second cook? This affects how strong the reliability claim can be.
- **Cuisine scope for the flavor reasoner:** ship cuisine-aware caveats for all cuisines, or restrict v0 demo dishes to the Western subset where FlavorGraph's signal is least contested (and say so)?
- **Safety-gate failure mode:** hard *block* (proposal never reaches the human) vs. *block-with-override-on-citation* — which for the n=5 demo? The culinary scout leans block-or-require-citation; pick one and document it.
- **Wk-0 user test:** will you run the multi-iteration/multi-day protocol (product §5) before heavy build, and is a 0/5 or "either way" result a real go/no-go you'll honor?
- **Live retrieval as P1:** is the eval explicitly designed to ship and run a complete result on 2 arms (ungrounded vs FlavorGraph+USDA/FoodOn) so a literature-retrieval slip never blocks the hero?
